# Implementation Requirements
1) Hardware Requirements
본 시스템은 웹 브라우저를 통해 실행되므로 일반적인 웹 사이트 접속이 가능한 수준의 PC 사양이면 모두 구동 가능하다. 단, 소리 녹음 및 청취 기능을 위해 아래의 하드웨어 장치를 요구한다.

마이크 (Audio Input): 사용자의 목소리를 녹음하기 위한 장치 (PC 내장 마이크 등)

스피커/이어폰 (Audio Output): 재생되는 루프 사운드를 듣기 위한 오디오 출력 장치

2) Software Requirements
운영체제 (OS): 웹 브라우저 실행이 가능한 모든 환경 (Windows, macOS 등)

실행 환경: HTML5 웹 표준 API(Web Audio API)를 지원하는 웹 브라우저 (Google Chrome 등)

개발 언어: JavaScript, HTML5, CSS3

3) Nonfunctional requirements (비기능적 요구사항)
서버 및 데이터베이스 제약: 본 시스템은 별도의 외부 서버나 외부 데이터베이스(RDBMS)를 사용하지 않는다. 시스템 내부에서 가상 서버 역할을 하는 Server 클래스가 설계되어 있으며, 회원 정보와 프로젝트 데이터는 이 클래스 안의 멤버 변수에 임시 저장되어 관리된다.

텍스트 파일 데이터 관리: 데이터베이스를 대신하여 프로젝트 폴더 내의 텍스트 파일(.txt 또는 .json)이 데이터 저장소 역할을 한다. 시스템 내부에서 자동으로 파일이 읽고 써지지만, 만약 사용자가 이 파일을 메모장 등으로 수동 편집할 경우에는 기존 예시 데이터의 양식 구조를 그대로 유지해야 한다. 다르게 입력할 경우 오류가 발생하여 프로그램이 멈출 수 있으므로 유의해야 한다.

마이크 권한 예외 처리: 프로그램 실행 시 브라우저의 마이크 접근 권한 승인이 필요하다. 사용자가 권한을 거부할 경우, 시스템 오류 방지를 위해 내부적으로 예외 처리를 수행하고 화면의 녹음 관련 UI 버튼들을 비활성화(disabled) 상태로 제어한다.
