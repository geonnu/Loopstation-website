# Implementation Requirements

1) Hardware Requirements
본 시스템은 기본적으로 웹 브라우저를 통해 실행되므로 웹 사이트 접속이 가능한 환경이라면 어디서든 동작한다. 다만, 브라우저 내부에서 실시간 다중 오디오 버퍼 연산 및 이펙터 처리를 직접 수행하므로 원활한 오디오 출력을 위해 아래와 같은 최소 사양을 요구한다.

CPU: INTEL Pentium IV 이상 (또는 웹 브라우징이 가능한 사양의 프로세서)

RAM: 1GByte 이상 (실시간 오디오 데이터 적재를 위한 최소 메모리 공간)

HDD or SSD: 10GByte 이상의 여유 공간 (웹 캐시 공간 및 프로젝트 설정 파일 보관용)

Audio Input: 사용자의 목소리 및 사운드 입력을 처리하기 위한 마이크 장치 필수 (PC 내장 마이크 혹은 외부 입력 장치)

Audio Output: 재생되는 다중 트랙 루프 사운드를 확인하기 위한 스피커 또는 이어폰 장치 필수

Network: 본 시스템은 모든 오디오 그래프 연산과 저장을 로컬 자원 기반으로 수행하므로, 최초 사이트 로딩 이후에는 네트워크가 연결되어 있지 않아도 오디오 녹음/재생 기능이 독립적으로 동작 가능함.

2) Software Requirements
Operating System: Windows 7 이상, macOS 등 웹 표준 브라우저 실행이 가능한 OS 환경

Implementation Language & Tech: JavaScript (ES6+ 이상 버전) 및 HTML5 Web Audio API 표준 스펙 명세 기반 규격

3) Nonfunctional Requirements (비기능적 요구사항)
서버 및 데이터베이스 구조 제약: 본 시스템은 별도의 외부 웹 호스팅 서버나 관계형 데이터베이스(RDBMS)를 연동하지 않는다. 대신 시스템 내부에서 가상 서버 및 데이터 관리 역할을 수행하는 Server 클래스가 설계되어 있으며, 회원 데이터나 루프 프로젝트 데이터 관리를 처리하기 위한 메모리 변수들이 이 Server 클래스 안에 멤버 변수 형태로 선언되어 동작한다.

로컬 파일 기반 스토리지 자원: 데이터베이스 테이블 구조 대신 로컬 프로젝트 폴더 내부에 위치한 구조화된 텍스트 파일(예: .json 또는 .txt)이 영속성 스토리지 역할을 대체한다. 이 파일 내부에는 테스트용 회원 계정 데이터와 루프 프로젝트 구조값(BPM 설정값, 개별 트랙 식별 데이터, 이펙터 세팅 파라미터)이 예시 데이터 형태로 채워져 들어간다.

수동 데이터 수정 시 유의사항: 저장소 역할을 하는 텍스트 파일 안에는 최초 구동 및 기능 검증을 위해 개발자가 임의로 작성한 예시 데이터가 미리 들어가 있다. 사용자가 프로그램 UI를 통해 회원가입을 하거나 프로젝트 저장 기능을 실행하면 프로그램 내부 로직에 의해 규격에 맞게 텍스트가 자동 갱신된다. 만약 사용자가 이 텍스트 파일을 메모장 등으로 수동 편집할 경우에는 기존 예시에 채워진 데이터 구조(Key-Value 규칙 및 구분자 등)를 반드시 똑같이 준수하여 입력해야 한다. 만약 양식과 다르게 임의로 입력할 경우 프로그램 구동 시 텍스트 파싱 오류가 발생하여 프로그램이 비정상 종료되거나 기능을 정상적으로 실행할 수 없으므로 편집 시 각별히 유의해야 한다.
