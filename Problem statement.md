5. Problem statement
이 장에서는 만들고자 하는 루프스테이션 시스템을 다양한 관점에서 생각한 결과를 기술한다. 루프 기반 음악을 웹 환경에서 구현할 때 발생할 수 있는 여러 문제들을 고려하였다.

첫 번째는 루프 녹음 방식의 한계이다. 일반적인 루프스테이션 장비에서는 하나의 트랙에 여러 번 녹음을 추가하여 소리를 계속 쌓아가는 overdub 기능을 지원한다. 이를 통해 하나의 트랙 안에서도 다양한 소리를 겹쳐 표현할 수 있다. 그러나 본 시스템에서는 하나의 트랙에 한 번의 녹음만 저장할 수 있도록 설계되어 있기 때문에 여러 소리를 같은 트랙에 겹쳐 녹음하는 기능을 제공하지 못한다. 따라서 복잡한 루프 구성을 만들기 위해서는 여러 트랙을 활용해야 하는 제약이 존재한다.

두 번째는 루프의 동기화 문제이다. 루프 기반 음악은 일정한 템포를 기준으로 반복되는 구조를 가지기 때문에 여러 개의 트랙이 동시에 재생될 경우 타이밍이 맞지 않으면 전체 음악의 흐름이 깨질 수 있다. 특히 여러 트랙을 동시에 사용하는 상황에서는 모든 트랙이 동일한 BPM 기준으로 정확하게 동기화되어 재생되는 것이 중요하다.

세 번째는 웹 환경에서 발생할 수 있는 오디오 처리 지연 문제이다. 웹 브라우저에서 오디오를 처리하는 과정에서 입력이나 재생 지연이 발생할 수 있으며, 이러한 지연은 루프 녹음이나 재생 과정에서 사용자 경험에 영향을 줄 수 있다.

네 번째는 마이크 입력 기능과 관련된 문제이다. 루프를 녹음하기 위해서는 브라우저의 마이크 권한을 허용해야 한다. 사용자가 마이크 사용을 허용하지 않을 경우 녹음 기능을 사용할 수 없는 상황이 발생할 수 있다.

다섯 번째는 웹 브라우저 환경에서의 안정성 문제이다. 브라우저마다 오디오 처리 방식이나 성능이 다를 수 있기 때문에 시스템이 다양한 웹 브라우저 환경에서도 안정적으로 동작하도록 고려해야 한다.

이와 같은 문제들을 항상 염두에 두면서 시스템을 설계해야 한다. 다음은 non functional requirement에 대한 내용이다. 후에 소프트웨어의 성능 요구 조건을 달성했는지 판단하기 위한 것을 목적으로 한다. 다음은 이 프로젝트의 NFS들이다.

➀ 여러 트랙의 루프는 사용자가 설정한 BPM 기준으로 정확하게 동기화되어 재생되어야 한다.  
➁ 사용자가 재생 또는 정지 버튼을 눌렀을 때 루프 제어 기능은 지연 없이 반응해야 한다.  
➂ 여러 트랙이 동시에 재생되는 상황에서도 시스템은 안정적으로 동작해야 한다.  
➃ 마이크 녹음 기능은 브라우저의 마이크 권한 요청을 통해 사용자의 허용을 받은 경우에만 동작해야 한다.  
➄ 시스템은 일반적으로 사용되는 웹 브라우저 환경에서 정상적으로 동작해야 한다.
