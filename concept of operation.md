4. Concept of Operation

1. 회원가입
| Purpose | 사용자가 시스템을 이용하기 위해 계정을 생성할 수 있도록 한다. |
| Approach | 사용자가 회원가입 화면에서 아이디와 비밀번호 등의 정보를 입력하여 계정을 생성할 수 있도록 한다. |
| Dynamics | 시스템을 처음 이용하는 사용자가 계정이 없는 경우 회원가입을 진행한다. |
| Goals | 사용자가 자신의 계정을 생성하여 시스템을 이용할 수 있도록 한다. |

2. 로그인
| Purpose | 사용자가 자신의 계정으로 로그인하여 개인 작업 기록을 관리할 수 있도록 한다. |
| Approach | 사용자가 아이디와 비밀번호를 입력하면 사용자 데이터베이스에서 정보를 조회하여 인증을 수행한다. |
| Dynamics | 사용자가 로그인하면 이전에 저장한 루프 프로젝트를 조회하거나 불러와 작업을 이어서 진행할 수 있다. |
| Goals | 사용자 계정을 기반으로 루프 프로젝트를 관리할 수 있도록 로그인 기능을 제공한다. |

3. 마이크 입력 활성화
| Purpose | 사용자가 마이크를 이용하여 사운드를 직접 녹음할 수 있도록 한다. |
| Approach | 시스템이 브라우저의 마이크 권한 요청 기능을 통해 사용자에게 마이크 사용 여부를 확인한다. |
| Dynamics | 사용자가 마이크 사용을 허용하면 시스템에서 마이크 입력을 받아 사운드를 녹음할 수 있다. |
| Goals | 사용자가 직접 사운드를 녹음하여 루프 제작에 활용할 수 있도록 한다. |

4. 루프 생성
| Purpose | 사용자가 사운드를 반복하여 루프 기반 음악을 생성할 수 있도록 한다. |
| Approach | 사용자가 사운드 라이브러리의 사운드를 재생하거나 마이크로 녹음한 사운드를 이용하여 루프를 생성할 수 있도록 한다. |
| Dynamics | 사용자는 여러 개의 트랙을 이용하여 사운드를 녹음하고 반복시켜 루프를 구성하며, 각 트랙은 루프 형태로 반복 재생될 수 있다. |
| Goals | 사용자가 루프 기반 음악을 생성할 수 있도록 루프 생성 기능을 제공한다. |

5. 루프 재생 제어
| Purpose | 사용자가 생성한 루프의 재생 상태를 제어할 수 있도록 한다. |
| Approach | 사용자가 각 트랙의 버튼을 이용하여 루프를 재생하거나 정지할 수 있으며, All Start와 All Stop 기능을 통해 모든 트랙을 동시에 제어할 수 있도록 한다. |
| Dynamics | 루프의 재생은 사용자가 설정한 BPM을 기준으로 진행되며, 재생 버튼을 누르면 해당 트랙의 루프가 BPM에 맞춰 반복 재생된다. 또한 All Start와 All Stop 기능을 통해 모든 트랙의 루프를 동시에 재생하거나 정지할 수 있다. |
| Goals | 사용자가 루프의 재생 상태를 쉽게 관리할 수 있도록 한다. |

6. BPM 설정
| Purpose | 루프의 템포를 설정하여 음악의 속도를 조절할 수 있도록 한다. |
| Approach | 사용자가 BPM 값을 설정하거나 변경하여 루프의 재생 속도를 조절할 수 있도록 한다. |
| Dynamics | 사용자가 BPM 값을 변경하면 이후 루프의 녹음 및 재생은 해당 BPM을 기준으로 진행된다. |
| Goals | 사용자가 루프의 템포를 자유롭게 설정할 수 있도록 한다. |

7. 효과 적용
| Purpose | 루프 사운드에 다양한 오디오 효과를 적용하여 사운드를 조정할 수 있도록 한다. |
| Approach | 사용자가 Reverb, EQ 등의 오디오 효과를 선택하여 루프에 적용할 수 있도록 한다. |
| Dynamics | 사용자는 루프 생성 또는 재생 과정에서 효과를 적용하여 사운드를 조정할 수 있다. |
| Goals | 사용자가 루프 사운드를 원하는 방식으로 조정할 수 있도록 한다. |

8. 루프 프로젝트 저장
| Purpose | 사용자가 생성한 루프 프로젝트를 저장하여 이후에도 다시 사용할 수 있도록 한다. |
| Approach | 루프의 트랙 구성, BPM, 효과 설정 등의 정보를 시스템에 저장한다. |
| Dynamics | 사용자가 작업을 완료하거나 중간 저장을 할 경우 루프 프로젝트를 저장할 수 있다. |
| Goals | 사용자가 생성한 루프 프로젝트를 저장하고 관리할 수 있도록 한다. |

9. 루프 프로젝트 불러오기
| Purpose | 이전에 저장한 루프 프로젝트를 다시 불러와 작업을 이어서 진행할 수 있도록 한다. |
| Approach | 로그인한 사용자가 자신의 계정에 저장된 루프 프로젝트 목록을 조회하고 선택하여 불러올 수 있도록 한다. |
| Dynamics | 사용자가 이전 작업을 수정하거나 이어서 작업할 경우 저장된 프로젝트를 불러온다. |
| Goals | 저장된 루프 프로젝트를 다시 사용할 수 있도록 한다. |

10. 설정 파일 공유
| Purpose | 루프 프로젝트의 설정을 파일 형태로 공유할 수 있도록 한다. |
| Approach | 루프 프로젝트의 BPM, 트랙 구성, 사용한 사운드 정보, 효과 설정 등을 파일로 내보내거나 업로드할 수 있도록 한다. |
| Dynamics | 사용자는 생성한 루프 프로젝트의 설정 파일을 다른 사용자와 공유하거나 해당 파일을 업로드하여 동일한 설정을 불러올 수 있다. |
| Goals | 루프 프로젝트 설정을 공유하고 재사용할 수 있도록 한다. |
